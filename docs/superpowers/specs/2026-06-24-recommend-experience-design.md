# 추천 경험 정리 + 카테고리 추첨 연출 통일

작성일: 2026-06-24
대상: `public/index.html`, `public/app.js`, `public/style.css`

## 배경 / 문제

카테고리 기능을 추가하면서 "추천" 경험이 애매해졌다.

- 추천 동작이 두 종류인데(식당 추첨 / 카테고리 추첨) 위계가 안 맞는다. 카테고리
  추첨은 작은 칩 버튼, 식당 추첨(랜덤 추천)은 큰 메인 버튼이라 형제처럼 보이지
  않는다.
- 카테고리 추첨은 슬롯머신·별 터짐 같은 연출 없이 토스트로 결과만 "띡" 띄우고
  끝나서 식당 추첨에 비해 초라하다.
- 카테고리 드롭다운이 추천 영역에 끼어 있어 시선이 분산된다. 드롭다운은 성격상
  "추천"이 아니라 목록을 좁히는 "필터"다.

## 목표

사용자가 원했던 두 가지 추천을 명확한 동급 동작으로 만들고, 둘 다 동일한 추첨
연출을 공유하게 한다.

- **카테고리 안에서 식당 추천** — 현재 필터가 걸린 풀에서 식당 하나를 뽑는다.
- **카테고리 자체 추천** — 식당이 2곳 이상인 카테고리 중 하나를 뽑는다.

## 설계 결정 (확정)

1. 두 추천을 **동급 두 버튼**으로 나란히 배치한다.
   - `🎲 뭐 먹을까` — 카테고리(메뉴 종류)를 뽑는다.
   - `🍽 어디 갈까` — 식당을 뽑는다.
2. 두 버튼은 **동일한 슬롯머신 + 별 터짐 연출**을 쓰고, **같은 결과 자리
   (`#random-result`)를 공유**한다.
3. **카테고리 드롭다운을 추천 영역에서 빼서** 검색창과 함께 목록 위 "필터 영역"
   으로 옮긴다. 추천 영역에는 두 버튼과 결과만 남는다.
4. 카테고리 추첨(`뭐 먹을까`)은 **카테고리만 뽑고 멈춘다**(2단계 연쇄 아님).
   대신 결과 카테고리로 **아래 드롭다운(필터)을 동기화**해 목록이 그 카테고리로
   필터된다. 식당까지 정하려면 사용자가 `어디 갈까`를 한 번 더 누른다.

## 화면 구조

```
오늘 뭐 먹지?

┌─ 추천 영역 (.recommend) ──────────────┐
│  [ 🎲 뭐 먹을까 ]   [ 🍽 어디 갈까 ]      │
│         오늘은 → ○○○   (슬롯+별, 공유)     │
│         <recommend-scope 안내문>          │
└────────────────────────────────────────┘

[ 식당 추가 폼 (#add-form) ]

┌─ 필터 영역 ──────────────────────────┐
│  [ 🔍 식당·메뉴 검색 (#search) ]          │
│  [ 전체 ▾ ] (#category-filter)          │
└────────────────────────────────────────┘

[ 식당 목록 (#menu-list) ]
```

> 추가 폼 위치는 현행 유지(추천과 필터 사이). 이번 작업 범위는 추천 경험과
> 카테고리 컨트롤 재배치이며, 추가 폼 이동은 별도 과제로 남긴다.

## 동작 상세

### `🍽 어디 갈까` (식당 추첨)

현행 동작 유지. 현재 필터(검색어 + 드롭다운에서 고른 카테고리)가 적용된 풀
(`currentPool`)에서 식당 하나를 슬롯으로 뽑는다.

- 결과: `오늘은 → 김밥천국`
- 당첨 식당의 "위치 보기" 컨트롤을 결과 아래에 붙인다(현행 유지).
- 풀이 2곳 미만이면 버튼 비활성(현행 `updateRecommendState` 유지).

### `🎲 뭐 먹을까` (카테고리 추첨)

토스트 방식을 버리고 슬롯 연출로 교체한다.

- 후보 산정: 검색이 적용된 목록(`filterMenus(allMenus, search)`)에서 식당이
  **2곳 이상인 카테고리**만 후보로 삼는다(현행 `categoryRandomBtn` 로직 유지).
- 후보 카테고리들을 슬롯머신으로 돌린다. 슬롯에 흘러가는 텍스트는 식당 이름이
  아니라 카테고리명("한식/중식/일식…")이다.
- 멈추면 `오늘은 → 한식` + 별 터짐(식당 추첨과 동일 연출).
- 동시에 `#category-filter` 값을 당첨 카테고리로 설정하고 `selectedCategory`를
  갱신한 뒤 `renderFiltered()`로 목록을 그 카테고리로 필터한다.
- 후보가 하나도 없으면 버튼 비활성(아래 상태 규칙 참고).

### 공유 슬롯 연출

현재 `randomBtn` 핸들러 안에 인라인으로 있는 슬롯머신 + `burstStars` 로직을
"표시할 항목 배열과 당첨 항목"을 받는 함수로 추출해 두 버튼이 공유한다.

- 입력: 슬롯에 흘릴 라벨 배열, 최종 당첨 라벨, (선택) 당첨 후 후처리 콜백.
- `prefers-reduced-motion`이면 슬롯·별 생략하고 결과만 표시(현행 유지).
- `winner` 클래스 재시작(reflow 트리거)으로 연속 추첨 시에도 매번 재생(현행 유지).

## 상태 / 엣지 케이스

- `어디 갈까`: `currentPool` 2곳 미만이면 비활성(현행).
- `뭐 먹을까`: 2곳 이상인 카테고리가 하나도 없으면 비활성.
- 비활성 사유는 두 버튼 아래 공유 안내문(`#recommend-scope`)에 표시한다.
  - 예) 식당이 1곳뿐: "식당이 1곳뿐이에요"(현행 문구 유지)
  - 예) 추첨 가능한 메뉴 종류 없음: "추첨할 메뉴 종류가 부족해요"
- 카테고리가 아직 백필 안 돼 후보 자체가 없으면(현행은 `category-controls`를
  숨김) `뭐 먹을까` 버튼도 비활성으로 둔다. 드롭다운(필터)은 필터 영역에서
  카테고리가 없을 때 숨김 처리(현행 `renderCategoryOptions`의 `hidden` 로직을
  드롭다운에만 적용).

## 코드 영향 범위

- `index.html`
  - `category-controls`(드롭다운)를 `.recommend` 밖, `#search-wrap` 아래로 이동.
  - 추천 영역에 두 버튼(`뭐 먹을까`, `어디 갈까`)을 나란히 배치. 기존
    `#category-random-btn`은 추천 영역으로 옮기고 라벨 변경, `#random-btn`
    라벨을 "어디 갈까"로 변경.
- `app.js`
  - 슬롯머신 + 별 연출을 공유 함수로 추출(현 `randomBtn` 핸들러 [592-653]).
  - `categoryRandomBtn` 핸들러를 토스트 → 슬롯 연출 + 드롭다운 동기화로 교체
    (현 [655-672]).
  - `updateRecommendState`에 `뭐 먹을까` 버튼 활성/비활성과 안내문 추가.
  - 카테고리 컨트롤 표시 로직(`renderCategoryOptions`)을 드롭다운/버튼 분리에
    맞게 조정.
- `style.css`
  - 두 추천 버튼을 동급으로 보이게 스타일 정리(현 `#random-btn` 스타일을 공유
    클래스로). 카테고리 드롭다운을 필터 영역 톤(검색창과 어울리게)으로 조정.

## 범위 밖 (YAGNI)

- 카테고리 → 식당 2단계 연쇄 추첨(사용자가 B안 선택).
- 식당 추가 폼 위치 이동, 삭제 버튼 위계 조정 등 별도 UX 개선.
- 위치 기반 실시간 추첨 전환(별도 로드맵, lunch-product-direction 참고).
